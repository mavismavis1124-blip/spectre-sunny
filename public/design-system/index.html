Design System Overview

The Spectre AI Design System provides a comprehensive set of design tokens, components, and patterns for building premium dark-themed trading and fintech applications.

üéØ Core Principles

  • Premium Dark Theme ‚Äî Deep, rich blacks with subtle gradients for a professional aesthetic
  • Glass Morphism ‚Äî Frosted glass effects with backdrop blur for depth and modern appeal
  • Smooth Animations ‚Äî Apple-like transitions using cubic-bezier easing functions
  • Visual Hierarchy ‚Äî Clear information architecture with proper contrast ratios
  • WOW Effects ‚Äî Subtle but impressive visual flourishes (glows, shimmers, pulses)
  • Institutional Grade ‚Äî Professional polish suitable for serious trading platforms

/* Helper: Format Large Numbers */
const formatLargeNumber = (num) => {
  if (num >= 1e9) return `$${(num / 1e9).toFixed(2)}B`
  if (num >= 1e6) return `$${(num / 1e6).toFixed(2)}M`
  if (num >= 1e3) return `$${(num / 1e3).toFixed(2)}K`
  return `$${num.toFixed(2)}`
}

/* Number Formatting Helpers */

// Format large numbers with K/M/B suffix
export const formatLargeNumber = (num) => {
  if (!num || isNaN(num)) return '$0'
  if (num >= 1e9) return `$${(num / 1e9).toFixed(2)}B`
  if (num >= 1e6) return `$${(num / 1e6).toFixed(2)}M`
  if (num >= 1e3) return `$${(num / 1e3).toFixed(2)}K`
  return `$${num.toFixed(2)}`
}

// Truncate address for display
const truncateAddress = (address) => {
  if (!address || address.length < 10) return address
  return `${address.slice(0, 6)}...${address.slice(-4)}`
}

// Format relative time
const formatAge = (date) => {
  const diffSecs = Math.floor((new Date() - date) / 1000)
  if (diffSecs < 60) return `${diffSecs}s ago`
  if (diffSecs < 3600) return `${Math.floor(diffSecs / 60)}m ago`
  if (diffSecs < 86400) return `${Math.floor(diffSecs / 3600)}h ago`
  return `${Math.floor(diffSecs / 86400)}d ago`
}

const KNOWN_TOKEN_COLORS = {
  'SPECTRE': '#8B5CF6', 'ETH': '#627EEA', 'BTC': '#F7931A',
  'SOL': '#9945FF', 'USDT': '#26A17B', 'USDC': '#2775CA',
  'PEPE': '#3D9E41', 'UNI': '#FF007A', 'LINK': '#2A5ADA',
}

// Generate color from string for unknown tokens
const generateColorFromString = (str) => {
  let hash = 0
  for (let i = 0; i < str.length; i++) {
    hash = ((hash << 5) - hash) + str.charCodeAt(i)
    hash = hash & hash
  }
  const hue = Math.abs(hash % 360)
  const saturation = 65 + (Math.abs(hash >> 8) % 20)
  const lightness = 50 + (Math.abs(hash >> 16) % 15)
  return `hsl(${hue}, ${saturation}%, ${lightness}%)`
}

// Get token color (known or generated)
const getTokenColor = (symbol) => {
  return KNOWN_TOKEN_COLORS[symbol?.toUpperCase()] 
    || generateColorFromString(symbol || 'default')
}

Best Practices

Guidelines for using the design system effectively.

‚úÖ Do's

  • Use CSS Variables ‚Äî Always use design tokens, never hardcode values
  • Consistent Spacing ‚Äî Use the spacing scale (--sp-*) for margins and padding
  • Proper Typography ‚Äî Use the type scale classes for text styling
  • Smooth Animations ‚Äî Apply transitions to all interactive elements
  • Glass Effects ‚Äî Use backdrop-filter for overlays and cards
  • Color Hierarchy ‚Äî Use proper text opacity levels for hierarchy
  • Responsive Design ‚Äî Test on multiple screen sizes

‚ùå Don'ts

  • Don't hardcode colors ‚Äî Always use CSS variables
  • Don't skip transitions ‚Äî All interactive elements should animate
  • Don't overuse glow ‚Äî Use glow effects sparingly for emphasis
  • Don't mix fonts ‚Äî Stick to the defined font families
  • Don't ignore contrast ‚Äî Ensure text is readable
  • Don't use light backgrounds ‚Äî This is a dark-first design system
